---
description: 要件定義ドキュメントを生成
---

ヒアリング内容をもとに要件定義ドキュメントを作成します。

## 入力
$ARGUMENTS
（/initでのヒアリング結果、または追加の要件）

## 生成するドキュメント

### docs/requirements.md

```markdown
# [プロジェクト名] 要件定義書

## 1. プロジェクト概要
### 1.1 目的
### 1.2 背景

## 2. ターゲットユーザー
### 2.1 ペルソナ
### 2.2 ユーザーの課題

## 3. 機能要件
### 3.1 MVP（必須機能）
| ID | 機能名 | 説明 | 優先度 |
|----|--------|------|--------|
| F-001 | | | 高 |

### 3.2 将来機能
| ID | 機能名 | 説明 | 優先度 |
|----|--------|------|--------|

## 4. 非機能要件
### 4.1 パフォーマンス
### 4.2 セキュリティ
### 4.3 可用性

## 5. ユースケース
### UC-001: [ユースケース名]
- アクター:
- 事前条件:
- 基本フロー:
- 代替フロー:
- 事後条件:

## 6. 用語集
| 用語 | 説明 |
|------|------|

## 7. 制約事項
```

## 手順
1. ヒアリング内容を整理
2. 上記フォーマットでドキュメント生成
3. docs/requirements.md として保存

---
完了したら `/design` で設計に進みます。
